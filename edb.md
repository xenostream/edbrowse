## Overview
이 프로그램은 언뜻 보기에 `/bin/ed` 를 재구현한 것처럼 보입니다. 실제로 몇 가지 ed 명령어를 입력해 보면, 자신이 이 프로그램을 실행하고 있다는 사실을 깨닫지 못할 수도 있습니다. 하지만, 계속 사용하다 보면 결국 edbrowse가 ed와 다른 점, 즉 “몇 가지 차이점” 을 발견하게 될 것입니다. 이에 대해서는 아래에서 설명하겠습니다.

ed를 다시 만드는 것은 완전히 시간 낭비처럼 보일 수 있습니다. 하지만, 이 프로그램이 “웹 브라우저 역할도 처리한다” 는 사실을 깨닫게 되면 이야기는 달라집니다. ed 내부에 내장된 웹 브라우저인 셈입니다. 마치 로컬 파일처럼 쉽게 URL을 편집할 수 있으며, 브라우즈 모드를 활성화하면 이와 같은 명령-응답(command-response) 프로그램에 적합한 방식으로 HTML 태그를 렌더링할 수 있습니다. 다시 말해, 대부분의 서식 정보는 제거하고 링크와 입력 양식만 유지합니다. 이를 통해 시각 장애 사용자도 음성이나 점자의 선형적 특성과 호환되는 애플리케이션을 통해 인터넷에 접속할 수 있습니다.

Edbrowse는 다음과 같은 기능을 제공합니다: 

- Edbrowse는 여러 파일을 동시에 편집하거나 여러 웹사이트를 동시에 탐색할 수 있으며, 잘라내기 및 붙여넣기와 유사한 방식으로 이들 간에 텍스트 블록을 전송할 수 있습니다. 예를 들어, `edbrowse *.c` 명령을 실행하면 현재 디렉토리에 있는 모든 C 소스 파일에 접근할 수 있습니다.

- ed의 정규 표현식은 좀 더 강력하고 유연한 Perl 호환 정규 표현식(PCRE)으로 대체되었습니다. 이를 통해 대용량 문서에서 텍스트를 찾거나 수정하는 작업이 훨씬 더 간편해집니다. 예를 들어, Perl의 “look ahead” 기능을 사용하면, foo 뒤에 느낌표나 물음표가 붙지 않은 한, foot 이나 food 가 아닌 foo 라는 단어 자체(대소문자 불문)를 bar 로 일괄 교체할 수 있습니다. 

```
ebre-
,s/\bfoo\b(?![!?])/bar/ig 
```

 정규 표현식 변경으로 인해 edbrowse는 ed와 100% 하위 호환되지는 않지만, 거의 비슷합니다.

- edbrowse는 http, https, ftp 또는 gopher 를 사용해서 현재 자신의 컴퓨터에서 파일을 여는 것만큼 쉽게 인터넷에서 웹 페이지를 불러올 수 있습니다. 페이지가 활성 버퍼로 불러오면, `b` 명령을 사용해서 페이지를 탐색하거나 텍스트로 표시할 수 있습니다. 이는 이미 ed에 포함되어 있던 다른 한 글자 명령어들을 보완합니다. 페이지를 열람한 후, 사용자는 하이퍼링크를 따라가거나 양식을 작성해서 제출할 수 있습니다.

`-fm` 옵션을 사용하면 edbrowse를 대화형 이메일 클라이언트로 전환할 수 있습니다. 이 옵션은 하나 이상의 POP3 계정에서 메일을 가져와 사용자에게 한 번에 하나씩 표시하며, 사용자는 이를 읽거나 삭제하거나 나중에 참조할 수 있도록 파일에 저장할 수 있습니다. 필터 기능을 사용하면 제목, 발신자 또는 수신자 이메일 주소를 기준으로 특정 이메일을 지정된 파일에 자동으로 저장할 수 있습니다. 다른 옵션을 사용하면 edbrowse가 SMTP를 통해 메일을 보낼 수 있습니다. IMAP도 지원됩니다.

- 데이터베이스 모드에서 edbrowse는 ODBC를 통해 SQL 테이블의 선택된 행과 열에 접근할 수 있습니다. edbrowse의 insert, substitute, delete 명령은 이제 각각 데이터베이스의 insert, update, delete 작업에 해당합니다. 이 기능은 일부 데이터베이스에서는 충분히 테스트되었으나, 다른 데이터베이스에서는 전혀 테스트되지 않았습니다. 텍스트 편집은 현재 버퍼의 범위를 벗어나 영향을 미치므로, 실행 취소(undo) 명령은 지원되지 않습니다. 따라서 행이 삭제되면 복구할 수 없이 영구히 사라집니다. 다행히 참조 무결성(referential integrity) 덕분에 이런 의도하지 않은 삭제가 방지되는 경우가 많습니다.

- 디렉터리를 편집할 때, 텍스트 각 줄은 하나의 파일에 해당합니다. 여기서도 substitute 명령어는 파일 이름을 변경하고, delete 명령어는 edbrowse 설정에 따라 파일을 삭제하거나 휴지통으로 이동시킵니다. 되돌리기(undo) 기능은 없으므로 파일을 휴지통으로 이동시키는 것이 좀 더 안전합니다. 하지만, 이렇게 해도 디스크 공간이 확보되지는 않습니다.

- edbrowse 설정 파일에는 리눅스의 .bashrc 파일과 유사한 함수(function)를 포함할 수 있습니다. 이런 함수는 다른 edbrowse 명령을 호출할 수 있으며, 해당 명령의 성공 여부에 따라 분기 및 루프를 수행할 수 있습니다. 이 간단한 예제로는 모든 x 를 y 로 변경합니다. 변경 사항이 있으면 파일은 디스크에 다시 기록됩니다. if 문에서 `*` 문자는 이전 명령이 성공했을 때만 참(true)이 됩니다.

```
function+xy {
  ,s/x/y/g
  if(*) {
          w
  }
}
```

현재 사용 중인 배포판에 edbrowse가 포함되어 있지 않다면, 기능이 다소 축소된 Perl 버전을 이용해서 Linux, Unix, Mac, Windows 등 어떤 컴퓨터에서든 곧바로 실행해 볼 수 있습니다. edbrowse.pl 을 사용해 보시고, 마음에 드신다면 git을 통해 패키지를 가져와 소스 코드를 직접 컴파일해서 모든 기능을 갖춘 C 버전을 실행해 보시기 바랍니다.

리눅스 사용자면서 사용 중인 배포판에 edbrowse가 포함되어 있지 않다면, 앞서 언급한 Perl 버전을 사용할 수도 있지만, 직접 빌드하는 번거로움 없이 모든 기능을 갖춘 C 버전을 실행할 수도 있습니다. 32비트 및 64비트 아키텍처용 “정적 링크 실행 파일” 은 edbrowse 홈페이지에 제공됩니다.

이 문서는 독자가 “ed에 익숙하다” 고 가정합니다. 사실 ed를 능숙하게 다룰 수 있다면 도움이 많이 됩니다. 인터넷 브라우저와 관련된 용어의 경험도 도움이 됩니다. 


## Other Languages 
먼저, 문자 집합(charsets)에 대해 간단히 설명하겠습니다. 영어는 한 바이트당 한 글자씩, 바이트 스트림에 쉽게 담을 수 있습니다. 사실 각 글자는 7비트로 표현되며, 여덟 번째 비트는 필요하지 않으므로 0 으로 설정됩니다. 이런 체계를 `ASCII` 라고 부르며, “영어 전용” 입니다.

인도유럽어족 언어들은 ñ (스페인어), è (프랑스어), ö (독일어) 같은 더 많은 문자를 포함합니다. ISO 8859-1 표준에 따라 여덟 번째 비트를 1 로 설정하면 이런 문자들도 여전히 단일 바이트 에 담을 수 있습니다. 체코어나 헝가리어 같은 언어들은 127 이상의 바이트에 다른 문자를 할당하는 ISO 8859-2 표준에 부합합니다. 또한 iso8859-3 등도 있습니다. 코드 페이지를 선택하면 해당 언어의 모든 문자는 여전히 1바이트에 들어갑니다. 이것이 바로 iso8859 표준이며, ASCII 와 하위 호환됩니다. 다시 말해, z 문자는 ASCII와 모든 iso8859-x 코드 페이지에서 122 이지만, 127 을 초과하는 더 높은 숫자들은 언어에 따라 다른 문자를 나타낼 수 있습니다.

한동안은 이런 방식이 잘 작동했지만, 만약 한 단락은 프랑스어로, 다른 단락은 체코어로 작성하고 싶다면 어떻게 해야 할까요? 글 중간에 iso8859-1 에서 iso8859-2 로 전환하거나, 모든 언어의 모든 문자를 동시에 표현할 수 있는 새로운 표준을 마련해야 합니다. 당연히 단일 바이트에 담을 수 없는 이런 문자 집합은 `utf8` 로 알려져 있으며, 현재 컴퓨터 산업 전반의 새로운 표준이 되었습니다. 소프트웨어는 사용자가 거주하는 국가에 따라 iso8859 페이지를 선택하고 숫자를 문자로 매핑할 필요가 없습니다. 대신, ñ 는 1바이트가 아닌 2바이트로 표현됩니다. 이는 사용자에게는 투명하게 처리됩니다. 화면에는 동일한 문자가 표시되며, 스크린 리더가 이 문자들을 음성 합성기로 전달할 경우 동일한 소리가 나옵니다. 현재 사용 중인 시스템이 어떤 것인지 확인하는 방법은 `$LANG` 을 출력하는 것입니다. 결과가 utf8 또는 utf-8 로 끝난다면(대부분 그럴 것입니다), 콘솔은 utf8 을 사용하고 있으며 2바이트 시퀀스를 기대합니다. 또한 파일에도 이런 기본 시퀀스가 포함되어 있을 가능성이 높습니다. utf8 이전의 구형 시스템은 각 문자를 단일 바이트로 저장하고, iso8859 코드 페이지를 사용해서 변환을 수행합니다.

문자 집합에 대해 좀 더 자세히 설명할 수 있지만, 여기서는 간략한 소개만 하겠습니다. edbrowse는 현재 `iso8859-1 ⇔ utf8` 변환만 지원하므로 더 이상 설명할 필요가 없습니다. 다른 iso8859 페이지들은 지원되지 않으며, utf8 이 보편화되었기 때문에 향후에는 iso8859-1 지원조차 중단될 수 있습니다. 

“검색 문자열을 찾을 수 없음” 같은 출력 및 오류 메시지는 국제화되었습니다. LANG 환경 변수를 설정하면 edbrowse를 모국어로 사용할 수 있습니다. 지원되는 언어는 아래와 같습니다. edbrowse를 다른 언어로 번역할 수 있다면 개발자에게 연락해 주시기 바랍니다. 

```
English: LANG=en (this is the default)
French: LANG=fr by Erwin Bliesenick including documentation
Brazilian Portuguese: LANG=pt_br by Cleverson Casarin Uliana
Polish: LANG=pl by Wojciech Gac
Russian: LANG=ru by Wojciech Gac
German: LANG=de by Sebastian Humenda
Italian: LANG=it by Enrico Mioso
Spanish: LANG=es by Oriol Prieto 
```

일부 edbrowse 메시지는 파일 열기, 읽기 또는 쓰기 같이 strerror 의 출력 내용을 포함합니다. 현재 사용 중인 언어에 맞는 libc 구성 모듈이 설치되어 있다면, 이런 텍스트 조각들도 국제화됩니다. 운영체제 설정 시 언어를 지정했다면, 해당 모듈은 이미 자동으로 설치되어 있을 가능성이 높습니다. 하지만, 다른 언어 모듈은 설치되지 않을 수 있습니다. `LANG=es_ES.UTF8` 를 사용하면 edbrowse 의 모든 메시지는 스페인어로 표시되지만, 사용자의 기본 언어가 독일어라서 libc 스페인어 모듈이 설치되지 않은 경우, LANG 변수에 해당하는 모듈이 없을 때 strerror 의 기본값인 영어로 strerror 부분이 표시될 수 있습니다. 이는 확실히 사용자에게 혼란을 줄 수 있습니다. 그렇긴 하지만, 대부분의 사용자는 운영체제의 언어와 일관되게 edbrowse를 사용할 것이므로 이런 문제는 발생하지 않을 것입니다.

출력이나 오류 메시지가 표시될 때, `$LANG` 에 utf8 또는 utf-8 문자열이 포함되지 않은 한, 악센트 부호 문자는 iso8859 페이지를 통해 단일 바이트로 출력됩니다. 이 경우 비 ASCII 문자는 utf8 을 사용해서 생성됩니다. 마찬가지로, 로컬 문서든 인터넷 웹사이트이든 버퍼의 내용은 `$LANG` 에 따라 단일 바이트 또는 다중 바이트 시퀀스로 표시됩니다. 현재는 utf8이 표준이 되었으며, edbrowse는 앞으로 계속 iso8859 페이지를 통해 텍스트나 오류 메시지를 표시하지는 않을 것임을 유의하시기 바랍니다. 즉, “iso8859는 더 이상 권장되지 않습니다.”

edbrowse에서 오래된 파일을 불러올 때, 즉 빈 버퍼에 파일을 읽어들이는 경우, 컴퓨터는 utf8로 설정되어 있는데 해당 파일이 iso8859 형식이라면, 파일을 보기 전에 실시간으로 변환됩니다. 따라서 사용자는 정상으로 표시됩니다. 이 기능을 비활성화하려면 `iu` 명령을 사용하시기 바랍니다. 동일한 파일에 데이터를 다시 쓸 때(예: 수정이나 추가를 한 경우), edbrowse는 해당 데이터를 원래의 단일 바이트 iso8859 문자 집합으로 다시 변환하며, 파일이 원래 DOS 텍스트 파일이었다면 DOS 줄바꿈(CRLF) 시퀀스를 복원합니다. 따라서 편집된 파일을 친구에게 다시 보내도, 친구가 기대하는 대로 그의 문자 집합으로 표시될 것입니다. 그러나 데이터 전체 또는 일부를 새 파일에 저장할 경우, 해당 데이터는 사용자의 컴퓨터에서 사용하는 문자 집합으로 유지됩니다.

이런 변환은 ZIP 파일, 실행 파일 또는 기타 형태의 이진 데이터에서는 절대 수행되서는 안 됩니다. “iso8859로 변환 중” 또는 “utf8로 변환 중” 이라는 메시지가 표시되는데 파일이 국제 문자 텍스트가 아닌 경우, 문제가 발생한 것입니다. 변환된 실행 파일을 실행하려고 시도하지 마시기 바랍니다. 제대로 작동하지 않을 것입니다.

사용 환경이 utf8인 경우, 검색 기능이 혼란을 야기할 수 있습니다. '소년' 을 뜻하는 스페인어 단어 niño 를 예로 들어 보겠습니다. ni.o 로 검색하면 이 텍스트 줄을 찾지 못할 수도 있습니다. 점 문자는 한 글자를 나타내며 ñ 과 일치해야 하지만, 이 악센트 부호가 붙은 글자는 사실 두 바이트를 차지합니다. 아이러니하게도 ni..o 로 검색해야만 원하는 내용을 찾을 수 있습니다. 말할 필요도 없이, 이는 매우 혼란스럽습니다. 

검색 및 치환 작업은 pcre 라이브러리를 통해 수행되며, 다행히 최신 버전은 utf8을 지원합니다. Edbrowse는 pcre에 특정 2바이트 문자열을 단일 문자로 처리하도록 지시하는 옵션을 전달하므로, 원하는 대로 동작합니다. 이제 ni.o 를 검색할 수 있게 되었습니다. utf8 검색 및 치환 기능을 일시적으로 비활성화하려면 `su8` 명령을 사용하시기 바랍니다.

일부 웹사이트는 여러 언어로 콘텐츠를 제공합니다. 예를 들어, twitter.com 은 영어 버전, 프랑스어 버전 등이 있습니다. 어떤 페이지를 요청할 때 언어를 선택할 수 있습니다. Edbrowse는 구성 파일의 `localizeweb` 키워드를 통해 이를 지원합니다. 예를 들어, .ebrc 설정 파일에 다음 항목을 추가하면 프랑스어 버전이 사용 가능한 경우 이를 선택하도록 지정됩니다.

```
localizeweb = fr
```

만약 지정하지 않으면, edbrowse는 거의 모든 시스템에 설정되어 있는 '$LANG` 환경 변수를 따릅니다. 따라서 구성 파일에 `localizeweb` 을 별도로 설정할 필요는 거의 없습니다. LANG이 설정되지 않으면 “기본값은 영어” 입니다. amazon.com 같은 일부 웹사이트는 언어 지정자가 없으면 제대로 작동하지 않기 때문에, edbrowse는 “항상 웹 서버에 언어 지정자를 전송” 합니다. 직접 `localizeweb` 을 설정하는 경우, 반드시 “언어 코드-하이픈-지역 코드 형식” 으로 설정해야 합니다.

# Chapter 2, Quick Reference Guide
