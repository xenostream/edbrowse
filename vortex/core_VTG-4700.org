* The Noobs Guide to Programming a Vortex Core - Aug 30, 2017
최근 들어 저는 기계식 키보드에 다소 병적인 집착을 보이게 되었는데, 제가 처음으로 손에 넣은 키보드 중 하나가 바로 악명 높은 =Vortex Core= 입니다. 잘 모르시는 분들을 위해 설명드리자면, Core는 40% 키보드, 즉 키가 단 47개뿐인 초소형 키보드입니다. 생김새는 다음과 같습니다:

![20260410_181858.jpg](images/20260410_181858.jpg)

Vortex Core.

(image taken from vortexgear.tw)

숫자 키도 없고! 기능 키도 없고! 방향키도 없다니! 도대체 어떻게 쓰는 거야????

Core를 활용해서 작업을 수행하는 데는 두 가지 요령이 있습니다:

- 두 개의 기능 키, ~Fn~ (파란색)과 ~Fn1~ (빨간색)입니다. 이 키들을 사용하면 키캡 앞면에 표시된 대로 좀 더 많은 키 조합을 사용할 수 있습니다(예: ~Fn1 + W~ 는 =F2= 키의 단축키입니다).

- 네 가지 기능 레이어. 표시된 모든 키와 조합은 키보드의 기본 레이어에 적용되지만, 거의 완전히 사용자 정의가 가능한 세 개의 레이어가 더 있습니다.

그리고 바로 이 Core의 레이어를 사용자 정의하고 재프로그래밍하는 방법에 대해 이 블로그 게시물에서 다루겠습니다.



** A word of warning
이 모든 작업을 수행하는 방법에 대한 설명은 기껏해야 부실할 뿐이며, 펌웨어 업데이트에 따라 변경될 가능성이 높습니다. (네, 이 키보드에는 펌웨어가 있습니다.) 저는 이 글을 쓰는 시점에 최신 펌웨어인 =1.04.03= 을 사용하고 있습니다.

이 글에 담긴 정보 대부분은 여러 레딧 게시글과 수많은 시행착오를 통해 얻은 것이므로, 제가 모르는 부분이나 틀린 내용이 약간 있을 수 있습니다. 그런 부분이 있다면 댓글로 알려주세요!

그런 점을 염두에 두고, 무엇을 할 수 있는지 살펴보겠습니다.


** Switching layers
키보드의 기본 레이어는 =레이어 0= 입니다. =Pn= 키와 왼쪽 스페이스바 사이에 색상 LED가 켜져 있지 않다면 현재 레이어 0에 있는 것입니다. 이 레이어는 전혀 사용자 정의할 수 없습니다!!

레이어를 변경하려면 다음과 같은 =Fn= 키 조합을 사용해야 합니다:

| 레이어   | 키 조합     | 표시                                    |
| 레이어 0 | Fn + M      | Pn 과 lSpace 사이에서 LED가 켜지지 않음 |
| 레이어 1 | Fn + ,      | Pn 과 lSpace 사이에서 빨간색 LED가 켜짐 |
| 레이어 2 | Fn + .      | Pn 과 lSpace 사이에서 녹색 LED가 켜짐   |
| 레이어 3 | Fn + rShift | Pn 과 lSpace 사이에서 파란색 LED가 켜짐 |

이 단축키들은 키캡 앞면에도 파란색으로 표시되어 있습니다.

기본적으로 이 모든 레이어는 똑같은 기능을 수행합니다. 즉, C 라고 표시된 키를 누르면 C 문자가 그대로 입력됩니다. 하지만, =레이어 1, 2, 3= 은 사용자 정의가 가능하기 때문에, C 키를 누르면 S, lCtrl, PgUp 등이 입력될 수도 있습니다. 바로 여기서 프로그래밍이 필요해집니다.

(한 가지 주의할 점은 레이어 수정 키(=Fn=, =M=, =,=, =.= 또는 =rShift=) 중 하나를 변경하면 단축키가 새로운 Fn 키를 사용하지만, 기존 수정 키는 그대로 유지된다는 것입니다. 예를 들어, Fn 을 lCtrl 위치로, M 을 B 위치로 옮긴다면, =레이어 0= 으로 전환하는 단축키는 Fn + M 이 아니라 lCtrl + M 이 됩니다. lCtrl + B 가 아닙니다.)

레이어 0 이외에 본인에게 가장 적합한 레이어를 선택한 다음, 사용자 지정을 시작하면 됩니다.


** Things that can’t be done
키보드 사용자 지정은 키 입력에 동작을 재할당하는 방식으로 이루어집니다. 예를 들어, Qwerty 대신 Colemak 키보드 배열을 사용하려면 모든 흰색 키를 Colemak 배열의 해당 키로 재할당해야 합니다. (E 대신 F, R 대신 P 등)

모든 흰색 키는 재할당할 수 있으며, 몇 가지 예외를 제외하고는 거의 모든 수정 키(베이지색 키)를 재할당할 수 있습니다.

- Fn1 은 전혀 이동할 수 없습니다.
- Pn 과 Fn 중 하나만 이동할 수 있으며, 둘 다 이동할 수는 없습니다. (단, 서로 위치를 바꾸는 경우는 예외입니다.)
- Fn1 을 사용하는 단축키는 수정할 수 없으며, Fn1 을 사용하는 새로운 단축키를 추가할 수도 없습니다.
- 레이어 수정키 단축키는 Fn 부분을 제외하고는 위에서 설명한 대로 수정할 수 없습니다.

왜 그럴까요? 아무도 모릅니다. 이는 그냥 우회해서 해결해야 할 제한 사항들입니다. 단일 키 입력과 Fn 단축키만으로 모든 유용한 기능을 구현할 수 있는 유연성은 여전히 충분하며, 그래도 부족하다면 여러 레이어가 있다는 점을 기억하시기 바랍니다.


** Enough talking, more programming.
좋습니다. 이제 중요한 부분입니다. 이 방법은 Fn 과 Pn 키를 제외한 모든 키의 기능을 재설정하는 데 사용됩니다.

모든 설정이 이루어지는 ‘프로그래밍 모드’ 로 들어가려면, Fn 키(또는 Fn 기능을 할당해 둔 키)와 rCtrl 이라고 표시된 키를 동시에 누릅니다. 그러면 두 스페이스 바 사이에 있는 파란색 LED가 켜집니다.

이제 재할당하려는 키(또는 단축키)를 누릅니다. 그러면 두 스페이스 바 사이의 LED가 빨간색으로 깜빡이기 시작합니다.

이제 앞서 입력한 키나 단축키에 할당할 레이어 0 키나 단축키를 누릅니다. 이 점은 매우 중요합니다!! 현재 레이어에 있는 키 조합이 아니라, 키캡에 표시된 라벨에 따른 키 자체를 지정해야 합니다. 이는 잠재적인 동작을 ‘잃어버리지’ 않기 위함입니다. 예를 들어, 현재 레이어의 Pg-Down 단축키를 덮어쓰더라도, 다른 단축키에 Pg-Down 을 할당할 수 있도록 남겨두어야 하기 때문입니다. 이 단계에서 LED는 계속 빨간색으로 깜빡입니다.

(LED가 빨간색으로 유지되는 이유는, 원한다면 T, G, B 에 표시된 시간 지연 기능을 사용해서 전체 매크로를 프로그래밍할 수 있기 때문입니다. 제 기억으로는 키/단축키를 누른 다음, 대기할 지연 시간을 입력하고, 다음 키/단축키를 누르는 식으로 진행되는 것으로 알고 있습니다.)

새로운 동작 입력을 모두 마치면, Pn (또는 Pn 에 할당된 키)을 눌러 변경 사항을 확인하고 저장합니다. 이제 LED는 파란색으로 고정됩니다.

추가 변경 사항을 계속 입력하거나, Fn + rCtrl 을 눌러 프로그래밍 모드를 종료할 수 있습니다.

언제든지 키보드를 15초 동안 건드리지 않으면 프로그래밍 모드는 자동으로 종료되고 일반 텍스트 입력 모드로 돌아갑니다.



*** For a real example…
흔히 사용되는 재프로그래밍 방법 중 하나는 키보드 오른쪽 하단, 익숙한 위치에 전용 방향키를 추가하는 것입니다. 이를 위해 rShift, rAlt, Menu, rCtrl 키를 사용할 수 있으며, 작동이 매우 원활합니다. 구체적인 설정 방법을 단계별로 설명해 드리겠습니다. 이 설명은 Fn/Pn 키가 사용자 정의되지 않았다고 가정합니다.

1. 사용자 지정 가능한 레이어를 선택합니다. 저는 파란색을 좋아해서(그리고 제 Pok3r 처럼 보라색 LED가 없기 때문에) 레이어 3을 사용합니다.
2. Fn + rCtrl 을 눌러 프로그래밍 모드로 들어갑니다. 스페이스 바 사이의 LED가 파란색으로 바뀝니다.
3. rShift 를 눌러 이 키를 재할당할 키로 지정합니다. LED는 빨간색으로 깜빡이기 시작합니다.
4. Fn + I 를 누릅니다. 이는 위쪽 화살표 키의 단축키입니다.
5. Pn 을 누릅니다. LED는 다시 파란색으로 바뀝니다.
6. rAlt 를 눌러 해당 키를 재할당합니다. LED는 빨간색으로 깜빡이기 시작합니다.
7. Fn + J 를 눌러 rAlt 에 왼쪽 화살표를 할당합니다.
8. Pn 을 다시 누릅니다. LED는 다시 파란색으로 바뀝니다.
9. Menu 를 눌러 해당 키를 재할당합니다. LED는 빨간색으로 깜빡이기 시작합니다.
10. Fn + K 를 눌러 Menu 에 아래쪽 화살표를 할당합니다.
11. Pn 을 다시 누릅니다. LED는 다시 파란색으로 바뀝니다.
12. rCtrl 키를 눌러 해당 키를 재할당합니다. LED는 빨간색으로 깜빡이기 시작합니다.
13. Fn + L 을 눌러 rCtrl 에 오른쪽 화살표를 할당합니다.
14. Pn 을 다시 누릅니다. LED는 파란색으로 돌아갑니다.
15. 이제 모든 설정이 완료되었으므로, Fn + rCtrl 을 눌러 프로그래밍 모드를 종료합니다.

자, 이제 멋지고 반짝이는 전용 화살표 키가 완성되었습니다!  (사실, 레이어 0 에서 좌측 Alt + Pn + rSpace 를 눌러도 됩니다.) 


*** But what about Fn and Pn?
위의 설정은 Fn 및 Pn 키에는 적용되지 않습니다. 이 키들은 별도의 특수 사용자 지정 단축키가 있기 때문입니다.

*참고* : 저는 항상 이 단계를 마지막에 처리합니다. 이유는 모르겠지만, 제가 진행한 모든 테스트에서 Fn 이나 Pn 키를 이동시킨 후에는 추가적인 재설정이 절반 정도만 작동하는 것으로 나타났습니다. 예를 들어, 현재 레이어에서는 Fn + S 에 # 을 할당할 수 있지만, Fn + A 에 @ 을 할당하려면 아무런 반응이 없습니다. 이유는 저도 모르겠습니다.

어쨌든...

Fn 과 Pn 중 하나를 다른 키(또는 서로)와 교체할 수 있는 옵션이 있습니다. 이 작업을 하려면, Fn 이라 표시된 버튼이나 Pn 이라 표시된 버튼(이동하려는 버튼) 중 하나를 누른 상태에서 lShift 키를 5초간 누르고 있으면 됩니다. 그러면 두 스페이스 바 사이에 있는 녹색 LED가 깜빡입니다. 그런 다음 교체하려는 키를 누르면 끝입니다!!

구체적인 예로, Fn 을 왼쪽 스페이스바로 바꾸려면(흔히 하는 방법 중 하나입니다) 다음과 같이 합니다:

1. 녹색 LED가 깜빡이기 시작할 때까지 Fn + lShift 를 누른 상태를 유지한 다음, 왼쪽 스페이스 바를 누릅니다.

2. 이제 왼쪽 스페이스 바는 Fn 키가 되고, Fn 이라 표시된 키는 두 번째 스페이스 바가 됩니다.

(두 키 중 하나만 이동할 수 있는 이유는, 이 과정을 반복할 경우(예: Fn 을 이동시킨 후 Pn 을 이동시키려고 하면) 첫 번째 변경 사항이 취소되기 때문입니다.)
