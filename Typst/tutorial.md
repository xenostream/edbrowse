# Tutorial
Typst 튜토리얼에 오신 것을 환영합니다! 이 튜토리얼은 Typst에서 문서를 작성하고 서식을 지정하는 방법에 대해 배우게 됩니다. 일상적인 작업부터 시작해서 점차 고급 기능을 소개합니다. 본 튜토리얼은 Typst, 다른 마크업 언어 또는 프로그래밍에 대한 사전 지식을 가정하지 않습니다. 단지 텍스트 파일 편집 방법은 알고 있다고 가정합니다.

시작하기 가장 좋은 방법은 [Typst](https://typst.app) 앱에 무료 가입한 후 아래의 단계를 따라가는 것입니다. Typst 앱은 실시간 미리보기, 구문 강조 표시 및 유용한 자동 완성 기능을 제공합니다. 또는 오픈소스 CLI를 사용해서 로컬 텍스트 편집기에서 따라할 수도 있습니다.

## When to use Typst
시작하기 전에 Typst가 무엇이며 언제 사용하는지 살펴보겠습니다. **Typst는 문서 조판을 위한 마크업 언어** 입니다. 또한 배우기 쉽고 빠르며 다용도로 설계되었습니다. Typst는 마크업이 포함된 텍스트 파일을 입력 받아 PDF 파일로 출력합니다.

Typst는 에세이, 기사, 학술 논문, 책, 보고서, 숙제 과제 등 장문의 텍스트를 작성하는 데 적합한 선택입니다. 또한 수학, 물리학, 공학 분야 논문처럼 수학 기호가 포함된 문서도 매우 잘 어울립니다. 마지막으로 강력한 스타일링 및 자동화 기능 덕분에 책이나 시리즈처럼 공통된 스타일을 공유하는 문서 집합에 탁월한 선택입니다.


## What you will learn
이 튜토리얼은 네 개의 장으로 구성되어 있습니다. 각 장은 이전 장의 내용을 바탕으로 진행됩니다. 각 장에서 배울 내용은 다음과 같습니다:

1. **Typst에서 글쓰기**: 텍스트 작성 방법과 이미지, 방정식 및 기타 요소의 삽입 방법을 배웁니다.
1. **서식 설정**: 문서의 서식 설정 방법을 배웁니다. 글꼴 크기, 제목 스타일 등을 포함합니다.
1. **고급 스타일링**: 저자 목록 및 중간 제목 같은 타이포그래픽 기능을 갖춘 과학 논문의 복잡한 페이지 레이아웃 생성 등을 배웁니다.
1. **템플릿 만들기**: 이전 장에서 작성한 문서를 바탕으로 재사용 가능한 템플릿을 구축합니다.

Typst를 즐겨주시길 바랍니다!

# Writing in Typst
시작하겠습니다! 대학에서 "기술 보고서를 작성하라" 는 과제를 받았다고 가정합니다. 이 보고서는 산문, 수학 공식, 제목, 그리고 그림이 포함됩니다. 작업을 시작하려면 Typst 앱에서 **새로운 프로젝트** 를 생성합니다. 그러면 편집기로 이동하며 두 개의 패널을 보게 됩니다: 문서를 작성하는 **소스 패널** 과 렌더링된 문서를 미리 보는 **미리보기 패널** 입니다.

![](./images/img_20260114_153932.jpg)

이미 보고서를 작성하기 위한 좋은 구상을 염두에 두고 있습니다. 그러니 서론부터 작성해 보겠습니다. 편집기 패널에 텍스트를 입력합니다. 입력한 텍스트는 즉시 미리보기 페이지에 표시되는 것을 확인할 수 있습니다.

```
In this report, we will explore the
various factors that influence fluid
dynamics in glaciers and how they
contribute to the formation and
behaviour of these natural structures.
```

이 튜토리얼 전체에 걸쳐 위와 같은 코드 예제를 보여드리겠습니다. 앱과 마찬가지로 첫 번째 패널의 마크업을 표시하고 두 번째 패널의 미리보기를 표시합니다. 예제를 보기 쉽게 페이지 크기를 축소했으니 무슨 일이 벌어지는지 확인할 수 있습니다.

다음 단계는 제목(heading)을 추가하고 일부 텍스트를 강조하는 것입니다. Typst는 가장 일반적인 서식 작업에 간단한 마크업을 사용합니다. 제목을 추가하려면 `=` 문자를 입력하고 이탤릭체로 텍스트를 강조하려면 해당 텍스트를 `_밑줄_` 로 묶습니다.

| 텍스트 패널 | 미리보기 패널 |
| --- | --- |
| ```
= Introduction
In this report, we will explore the
various factors that influence _fluid
dynamics_ in glaciers and how they
contribute to the formation and
behaviour of these natural structures.
``` |

| ![](./images/img_20260114_154035.jpg) |

아주 쉽네요! 새로운 단락(paragraph)을 추가하려면 텍스트 두 줄 사이에 빈 줄을 넣기만 하면 됩니다. 해당 단락에 소제목(subheading)이 필요하면 `=` 대신 `==` 를 입력합니다. `=` 문자의 개수가 제목의 중첩 수준을 결정합니다.

이제 빙하 역학에 영향을 미치는 몇 가지 상황을 나열하겠습니다. 이를 위해 **번호가 매겨진 목록** 을 사용합니다. 목록의 각 항목의 줄 시작 부분에 `+` 문자를 입력합니다. Typst는 항목에 자동으로 순차적으로 번호를 매깁니다.

```
+ The climate
+ The topography
+ The geology
```

![](./images/img_20260114_154108.jpg)

**글머리 기호 목록** 을 추가하려면 `+` 기호 대신 `-` 기호를 사용합니다. 또한 목록은 중첩할 수 있습니다: 예를 들어 위 목록의 첫 번째 항목에 들여쓰기를 하면 하위 목록을 추가할 수 있습니다.

```
+ The climate
  - Temperature
  - Precipitation
+ The topography
+ The geology
```

![](./images/img_20260114_154128.jpg)


## Adding a figure
보고서에 그림을 추가하면 보다 나아질 것 같나요? 이제 그림을 추가하겠습니다. Typst는 PNG, JPEG, GIF, SVG, PDF, WebP 형식의 이미지를 지원합니다. 프로젝트에 이미지 파일을 추가하려면 먼저 왼쪽 사이드바의 상자 아이콘을 클릭해서 **파일 패널** 을 엽니다. 여기서 프로젝트의 모든 파일 목록을 확인할 수 있습니다. 현재는 단 하나뿐입니다: 현재 작성 중인 **메인 Typst 파일** 입니다. 다른 파일을 업로드하려면 오른쪽 상단 모서리의 화살표 버튼을 클릭합니다. 그러면 업로드 대화 상자가 열리고 여기서 컴퓨터에서 업로드할 파일을 선택할 수 있습니다. 지금은 보고서에 사용할 이미지 파일을 선택합니다.

![](./images/img_20260114_154236.jpg)

이전에도 살펴봤듯이 특정 기호(**마크업** 이라 함)는 Typst에서 특별한 의미를 지닙니다. `=, -, +, _` 기호는 각각 제목(`heading`), 목록(`list, enum`), 강조 텍스트(`emph`)를 생성하는 데 사용할 수 있습니다. 그러나 문서에 삽입하려는 모든 요소에 이렇게 특수 기호를 할당하면 암호처럼 복잡해지고 다루기도 어려워질 것입니다. 따라서 Typst는 **가장 흔히 사용되는 요소** 들에 대해서만 마크업 기호를 예약합니다. 그 외의 **모든 요소는 함수를 통해 삽입** 됩니다. 따라서 보고서 페이지에 이미지를 표시하려면 Typst의 이미지(`image`) 함수를 사용해야 합니다.

```
#image("glacier.jpg")
```

![](./images/img_20260114_154301.jpg)

일반적으로 **함수(function)는 일련의 인자에 대한 출력을 생성** 합니다. 마크업 내에서 함수를 호출할 때 인자를 제공하면 Typst가 결과(함수의 반환값)를 문서에 삽입합니다. 지금 경우는 이미지 함수는 하나의 인자를 받습니다: 이미지 파일에 대한 경로입니다. 마크업 내에서 함수를 호출하려면 먼저 `#` 문자를 입력한 후 바로 함수 이름을 입력합니다. 그런 다음 괄호 안에 인수를 넣습니다. Typst는 함수의 인수 목록에 사용할 수 있는 다양한 데이터 유형을 인식합니다. 이 경우 파일 경로는 짧은 텍스트 문자열로 큰따옴표로 묶어야 합니다.

삽입된 이미지는 페이지 전체 너비를 사용합니다. 이를 변경하려면 이미지 함수에 너비(`width`) 인수를 전달합니다. 이것은 **명명된 인수로 `이름:값` 쌍으로 지정** 합니다. 인수가 여러 개일 경우는 쉼표로 구분하므로 먼저 경로 뒤에 쉼표를 추가합니다.

```
#image("glacier.jpg", width: 70%)
```

![](./images/img_20260114_154328.jpg)

`width` 인자는 상대적 길이입니다. 본 예제는 백분율로 지정해서 이미지가 페이지 너비의 70%를 차지하도록 설정했습니다. 1cm 나 0.7in 같은 절대값으로 지정할 수 있습니다.

텍스트와 마찬가지로 이미지도 이제 **기본적으로 페이지 왼쪽에 정렬** 됩니다. 또한 캡션도 없습니다. `figure` 함수를 사용해서 이를 수정하겠습니다. 이 함수는 그림의 내용을 **위치 매개변수** 로 선택적인 캡션은 **이름이 지정된 매개변수** 로 받습니다.

`figure` 함수의 인자 목록 내에서 Typst는 이미 **코드 모드** 에 있습니다. 이것은 이미지 함수 호출 앞에 사용한 해시(`#`) 문자를 제거함을 의미합니다. **해시 문자는 마크업 내에서 사용할 때만 필요** 합니다.(일반 텍스트와 함수 호출을 구분하기 위해...)

캡션은 임의의 마크업으로 구성됩니다. 함수에서 마크업을 사용하려면 대괄호(`[...]`)로 묶습니다. 이런 구조를 **콘텐츠 블록** 이라 합니다.

```
#figure(
  image("glacier.jpg", width: 70%),
  caption: [
    _Glaciers_ form an important part
    of the earth's climate system.
  ],
)
```

![](./images/img_20260114_154444.jpg)

보고서를 작성할 때 삽입한 그림을 참조하고 싶을 수 있습니다. 이를 위해 우선 그림에 **레이블을 지정** 합니다. 레이블은 문서 내의 요소를 고유하게 식별합니다. 이미지 함수 뒤에 각괄호(`<...>`)로 이름을 묶어 레이블을 추가합니다. 이후 본문에서 그림을 참조할 때는 `@` 기호 뒤에 해당 레이블을 입력하면 됩니다. 제목과 방정식도 레이블을 지정해서 참조할 수 있습니다.

```
Glaciers as the one shown in
@glaciers will cease to exist if
we don't take action soon!

#figure(
  image("glacier.jpg", width: 70%),
  caption: [
    _Glaciers_ form an important part
    of the earth's climate system.
  ],
) <glaciers>
```

![](./images/img_20260114_154733.jpg)

> [!NOTE] 지금까지는 함수에 콘텐츠 블록(대괄호 안의 마크업)과 문자열(따옴표 안의 텍스트)을 전달했습니다. 둘 다 텍스트를 포함하는 것처럼 보입니다. 그러면 차이점은 무엇일까요?
>
> 콘텐츠 블록은 텍스트 뿐만 아니라 다른 종류의 마크업, 함수 호출 등을 포함할 수 있는 반면, 문자열은 단순히 문자의 연속일 뿐 그 외에는 아무것도 아닙니다.
>
> 예를 들어, 이미지 함수는 이미지 파일의 경로를 기대합니다. 텍스트 단락이나 다른 이미지를 이미지 경로 매개변수로 전달한다는 것은 말이 되지 않습니다. 그래서 여기선 문자열만 허용됩니다. 반면, 텍스트는 유효한 콘텐츠 유형이므로 콘텐츠가 필요한 곳이라면 어디서나 문자열이 작동합니다.


## Adding a bibliography
보고서를 작성하려면 일부 주장을 뒷받침해야 합니다. 참고문헌 기능을 사용해서 문서에 참고문헌(`bibliography`) 목록을 추가할 수 있습니다. 이 기능은 참고문헌 파일의 경로를 인자로 입력받습니다.

Typst의 기본 참고문헌 형식은 `Hayagriva` 파일이지만 호환성을 위해 `BibLaTeX` 파일도 사용할 수 있습니다. 동료가 이미 문헌 조사를 수행해서 `works.bib` 파일을 보내주었으므로 해당 파일을 사용합니다. 파일 패널을 통해 참고문헌 파일을 업로드하면 Typst에서 해당 파일에 접근할 수 있습니다.

문서에 참고문헌 목록이 포함되면 해당 목록에서 인용 작업을 처리할 수 있습니다. **인용은 레이블 참조와 동일한 구문을 사용** 합니다. 출처를 처음 인용하는 즉시 해당 출처는 문서의 참고문헌 섹션에 표시됩니다. Typst는 다양한 인용 및 참고문헌 스타일을 지원합니다. 자세한 내용은 참고 자료를 참조하시기 바랍니다.

```
= Methods
We follow the glacier melting models
established in @glacier-melt.

#bibliography("works.bib")
```

## Maths
방법론을 구체화한 후에는 문서의 핵심인 수학식으로 넘어갑니다. Typst는 내장된 수식 편집 기능을 갖추고 **자체 수학 표기법을 사용** 합니다. 간단한 수학식부터 시작하겠습니다. 수식 표현임을 Typst에 알리기 위해 `$` 기호로 감쌉니다:

```
The equation $Q = rho A v + C$
defines the glacial flow rate.
```

![](./images/img_20260114_154927.jpg)

위의 수학식은 인라인으로 설정되어 주변 텍스트와 같은 줄에 표시됩니다. 별도의 줄에 표시하려면 수학식 시작과 끝 부분에 각각 빈 라인을 하나씩 삽입합니다:

```
The flow rate of a glacier is
defined by the following equation:

$ Q = rho A v + C $
```

![](./images/img_20260114_155015.jpg)

Typst는 수학식에 사용한 Q, A, v, C 같은 **단일 문자는 그대로 표시** 한 반면, `rho` 는 그리스 문자로 변환했습니다. 수학 모드에서 단일 문자는 항상 원문 그대로 표시합니다. 그러나 여러 문자는 기호, 변수 또는 함수명으로 해석합니다. 단일 문자 간 곱셈을 암시하려면 그 사이에 공백을 넣습니다.

또한 여러 글자로 구성된 문자열을 만들려면 따옴표로 묶으면 됩니다:

```
The flow rate of a glacier is given
by the following equation:

$ Q = rho A v + "time offset" $
```

![](./images/img_20260114_155054.jpg)

논문에는 합계 공식도 포함해야 합니다. 합계 기호를 사용한 후 아랫첨자와 윗첨자로 합계 범위를 명시할 수 있습니다:

```
Total displaced soil by glacial flow:

$ 7.32 beta +
  sum_(i=0)^nabla Q_i / 2 $
```

![](./images/img_20260114_155131.jpg)

기호나 문자열에 **`_` 문자를 입력한 후 아랫첨자를 입력** 합니다. 마찬가지로 **윗첨자를 추가하려면 `^` 문자를 사용** 합니다. 아랫첨자나 윗첨자가 여러 개로 구성될 경우는 괄호로 묶어야 합니다.

위의 예제는 분수를 삽입하는 방법도 보여줍니다: **분자와 분모 사이에 `/` 문자** 를 넣으면 Typst가 자동으로 분수로 변환합니다. 또한 괄호는 지능적으로 해석되므로 계산기에 입력하듯이 표현식을 작성하면 Typst가 **괄호로 묶인 하위 표현식을 적절한 표기법으로 대체** 합니다.

```
Total displaced soil by glacial flow:

$ 7.32 beta +
  sum_(i=0)^nabla
    (Q_i (a_i - epsilon)) / 2 $
```

![](./images/img_20260114_155156.jpg)

모든 수학 구조가 특별한 구문을 가지는 것은 아닙니다. 대신, 이전에 살펴본 이미지 함수 같은 함수를 사용합니다. 예를 들어, 열 벡터를 삽입하려면 `vec` 함수를 사용할 수 있습니다. 수학 모드 내에서 함수 호출은 `#` 문자로 시작할 필요가 없습니다.

```
$ v := vec(x_1, x_2, x_3) $
```

![](./images/img_20260114_155218.jpg)

일부 함수는 수식 모드 내에서만 사용할 수 있습니다. 예를 들어, `cal` 함수는 집합(set)에 흔히 사용되는 서예체 글자를 조판하는 데 사용됩니다. 참고 자료의 수식 섹션에는 수식 모드에서 사용할 수 있는 모든 함수의 완전한 목록을 제공합니다.

한 가지 더: 화살표 같은 기호는 다양한 변형이 존재합니다. 기호 이름 뒤에 점과 수정자 이름을 붙여 이러한 변형들 중에서 선택할 수 있습니다:

```
$ a arrow.squiggly b $
```

![](./images/img_20260114_155240.jpg)

이 표기법은 마크업 모드에서 사용할 수 있지만 심볼 이름 앞에 `#sym` 을 붙여야 합니다. 사용 가능한 모든 심볼 목록은 심볼 섹션을 참조하시기 바랍니다.


## Review
이제 Typst에서 기본적인 문서를 작성하는 방법에 대해 학습했습니다. 텍스트 강조, 목록 작성, 이미지 삽입, 내용 정렬, 수식 입력 방법을 배웠습니다. 또한 Typst의 함수도 알아봤습니다. Typst는 문서에 삽입할 수 있는 다양한 콘텐츠를 제공합니다. 예를 들어 표, 도형, 코드 블럭 등이 있습니다. 이러한 기능과 기타 기능에 대해 자세히 알아보려면 참조 자료를 살펴보시기 바랍니다.

# Formatting
지금까지 여러분은 텍스트와 몇 개의 수학식과 이미지를 포함한 보고서를 작성했습니다. 하지만 여전히 매우 평범해 보입니다. 조교는 여러분이 새로운 조판 시스템을 사용한 사실을 아직까지 모르고 여러분의 보고서가 다른 학생들의 제출물과 잘 어울리길 원합니다. 이 장에는 Typst의 스타일링 시스템을 사용해서 보고서를 포맷팅하는 방법을 살펴보겠습니다.


## Set rules
이전 장에서 살펴본 것 같이 Typst는 **콘텐츠를 삽입하는 함수**(예: `image` 함수)와 **인자로 받은 콘텐츠를 조작하는 함수**(예: `align` 함수)가 있습니다. 예를 들어, 글꼴을 변경할 때 가장 먼저 떠오른 생각은 해당 작업을 수행하는 **함수를 찾아 문서 전체를 그 함수로 감싸는 것** 일 수 있습니다.

```
#text(font: "New Computer Modern")[
  = Background
  In the case of glaciers, fluid
  dynamics principles can be used
  to understand how the movement
  and behaviour of the ice is
  influenced by factors such as
  temperature, pressure, and the
  presence of other fluids (such as
  water).
]
```

![](./images/img_20260115_051554.jpg)

잠깐만요. 함수의 모든 인자는 괄호 안에 명시해야 하지 않았나요? 왜 괄호 뒤에 내용이 담긴 두 번째 대괄호가 있을까요? 그 해답은 함수에 내용을 전달하는 작업이 Typst에선 매우 흔한 작업이기 때문에 이를 처리하기 위한 특별한 구문이 존재하기 때문입니다: 내용을 인수 목록 안에 넣는 대신, 인수 바로 뒤에 대괄호 안에 직접 작성할 수 있으므로 구두점을 절약할 수 있습니다.

위에서 보듯이 이 방법은 효과가 있습니다. 텍스트(`text`) 함수를 사용하면 그 안에 포함된 모든 텍스트의 글꼴을 조정할 수 있습니다. 그러나 문서를 수 많은 함수로 감싸고 스타일을 선택적으로 그리고 그 자리에서 직접 적용하는 것은 금방 번거로워질 수 있습니다.

다행히 Typst에는 좀 더 우아한 해결책이 있습니다. 세트 규칙(set rules)을 사용하면 특정 유형의 콘텐츠가 나타나는 모든 위치의 스타일 속성을 적용할 수 있습니다. 세트 규칙을 작성하려면 `set` 키워드를 입력한 후 속성을 설정하려는 함수 이름을 입력하고 괄호 안에 인자 목록을 추가합니다.

```
#set text(
  font: "New Computer Modern"
)

= Background
In the case of glaciers, fluid
dynamics principles can be used
to understand how the movement
and behaviour of the ice is
influenced by factors such as
temperature, pressure, and the
presence of other fluids (such as
water).
```

![](./images/img_20260115_051616.jpg)

 > [!NOTE] 여기서 무슨 일이 일어났는지 좀 더 기술적으로 알고 싶으신가요?
>
> 세트 규칙은 함수의 일부 매개변수에 대해 향후 해당 함수를 사용할 때마다 적용될 **기본값을 설정하는 것** 으로 개념화할 수 있습니다.


## The autocomplete panel
앱에서 몇 가지 기능을 따라 하셨다면 `#` 문자를 입력할 때마다 사용 가능한 함수를 보여주는 패널이 나타나고 인수 목록 내에는 사용 가능한 매개변수를 보여준다는 점을 눈치채셨을 겁니다. 이것이 바로 **자동 완성 패널**입니다. 문서를 작성할 때 매우 유용하게 사용할 수 있습니다: 제안된 내용을 적용하려면 리턴 키를 누르거나 화살표 키로 원하는 항목으로 이동합니다. `Esc` 키를 눌러 패널을 닫을 수 있으며 `#` 을 입력하거나 `Ctrl + Space` 를 눌러 다시 패널을 열 수 있습니다. 함수에 맞는 올바른 인수를 찾으려면 자동 완성 패널을 활용하시기 바랍니다. 또한 대부분의 제안은 해당 기능에 대한 간단한 설명이 함께 제공됩니다.

![](./images/img_20260115_051712.jpg)


## Set up the page
세트 규칙으로 돌아가서 규칙을 작성할 때는 스타일링하려는 요소의 유형에 따라 적절한 함수를 선택합니다. 세트 규칙에 흔히 사용하는 함수 목록은 다음과 같습니다:

- 텍스트의 글꼴 종류, 크기, 색상 및 기타 속성을 설정하는 텍스트(`text`)
- 페이지 크기, 여백, 머리글 설정, 열 사용 활성화 및 바닥글 설정 페이지(`page`)
- 단락 정렬, 줄 간격 설정 등을 처리하는 단락(`par`)
- 제목의 모양을 설정하고 페이지 번호를 활성화하는 제목(`heading`)
- PDF 출력물에 포함될 제목 및 저자 등의 메타데이터를 설정하는 문서(`document`)

모든 함수의 매개변수를 설정할 수 있는 것은 아닙니다. 일반적으로 함수가 어떻게 작업을 수행할지 지시하는 매개변수만 설정할 수 있으며 작업 대상이 무엇인지 지시하는 매개변수는 설정할 수 없습니다. 함수 참조 페이지에는 설정 가능한 매개변수가 모두 명시되어 있습니다.

이제 보고서에 몇 가지 스타일을 추가하겠습니다. 여백을 좀 더 넓히고 세리프 글꼴을 사용합니다. 예제를 위해 다른 페이지 크기도 설정합니다.

```
#set page(
  paper: "a6",
  margin: (x: 1.8cm, y: 1.5cm),
)
#set text(
  font: "New Computer Modern",
  size: 10pt
)
#set par(
  justify: true,
  leading: 0.52em,
)

= Introduction
In this report, we will explore the
various factors that influence fluid
dynamics in glaciers and how they
contribute to the formation and
behaviour of these natural structures.

...

#align(center + bottom)[
  #image("glacier.jpg", width: 70%)

  *Glaciers form an important
  part of the earth's climate
  system.*
]
```

![](./images/img_20260115_051756.jpg)

여기서 주목할만한 몇 가지 사항이 있습니다.

첫 번째는 페이지 세트 규칙(page set rule)입니다. 이 규칙은 두 개의 인수를 받습니다: 페이지 크기와 페이지 여백입니다. 페이지 크기는 문자열입니다. Typst는 수 많은 표준 페이지 크기를 지원하지만 사용자 정의 페이지 크기도 지정할 수 있습니다. 여백(`margin`)은 사전(dictionary) 형식으로 지정합니다. **사전은 키-값 쌍의 집합** 입니다. 이 경우 키는 x 와 y 이며 값은 각각 가로 및 세로 여백입니다. 또한 left, right, top, bottom 키를 가진 사전(dictionary)을 전달해서 각 면별로 별도의 여백을 지정할 수 있습니다.

다음은 텍스트 세트 규칙(text set rule)입니다. 여기선 글꼴 크기를 10pt 로 글꼴 패밀리는 “New Computer Modern” 으로 설정합니다. Typst 앱에는 문서에 사용할 수 있는 다양한 글꼴이 포함되어 있습니다. 텍스트 기능의 인수 목록에서 자동 완성 패널을 통해 현재 사용 가능한 글꼴을 확인할 수 있습니다.

또한 단락의 줄 간격(일명 `leading`)도 설정했습니다: 이것은 길이 값으로 지정하며 글꼴 크기에 상대적인 리딩으로 지정하기 위해 `em` 단위를 사용했습니다: 1em 은 현재 설정된 글꼴 크기(기본값은 11pt)와 동일합니다.

마지막으로 중앙 정렬에 수직 정렬을 추가해서 이미지를 하단 정렬했습니다. 수직 및 수평 정렬은 `+` 연산자를 사용해서 결합하면 2차원 정렬을 생성할 수 있습니다.


## A hint of sophistication
문서를 보다 명확하게 구성하기 위해 제목에 번호를 매기겠습니다. 이를 위해 제목(`heading`) 함수의 번호 매기기 매개변수를 설정합니다.

```
#set heading(numbering: "1.")

= Introduction
#lorem(10)

== Background
#lorem(12)

== Methods
#lorem(15)
```

![](./images/img_20260115_051825.jpg)

예제는 “1.” 문자열을 번호 매기기(`numbering`) 매개변수로 지정했습니다. 이것은 Typst에게 각 제목 레벨의 번호 사이에 점을 추가하고 아라비아 숫자 형태로 제목에 번호를 매기도록 지시합니다. 제목에 문자, 로마 숫자 및 기호를 사용할 수 있습니다:

```
#set heading(numbering: "1.a")

= Introduction
#lorem(10)

== Background
#lorem(12)

== Methods
#lorem(15)
```

![](./images/img_20260115_051851.jpg)

이 예제는 `lorem` 함수를 사용해서 일부 자리 표시자 텍스트를 생성합니다. 이 함수는 숫자를 인수로 받아 해당 수 만큼의 Lorem Ipsum 텍스트를 생성합니다.

> [!NOTE] 제목(`heading`)과 본문(`text`) 세트 규칙이 해당 기능으로 생성하지 않은 텍스트와 제목에도 적용되는 이유가 궁금하셨나요?
>
> Typst는 사용자가 `= Conclusion` 을 작성할 때마다 내부적으로 `heading` 함수를 호출합니다. 사실 함수 호출 즉, `#heading[Conclusion]` 은 위의 `heading` 마크업(`=`)과 동일합니다. 다른 마크업 요소들도 이와 유사하게 작동하며 해당 함수 호출을 위한 구문 설탕(Syntax sugar)에 불과합니다.


## Show rules
이젠 결과물에 상당히 만족하실 겁니다. 하지만 마지막으로 한 가지 수정할 사항이 있습니다: 현재 작성 중인 보고서는 좀 더 큰 프로젝트를 위한 것으로 그 프로젝트의 이름은 문장 내에서도 항상 로고와 함께 표시되야 합니다.

여러분은 선택지를 고려해야 합니다. 우선 검색 및 바꾸기 기능을 통해 로고가 사용된 모든 위치 앞에 `#image(“logo.svg”)` 함수 호출을 추가할 수 있습니다. 이것은 매우 지루합니다. 대신 항상 로고 이미지와 텍스트를 반환하는 **사용자 정의 함수를 정의** 할 수 있습니다. 그러나 좀 더 쉬운 방법이 있습니다:

쇼 규칙(show rules)을 사용하면 Typst가 **특정 요소를 표시하는 방식을 재정의** 할 수 있습니다. Typst가 다르게 표시할 요소와 그 요소의 모양을 지정합니다. 쇼 규칙은 텍스트 인스턴스, 다양한 함수, 심지어 문서 전체에도 적용할 수 있습니다.

```
#show "ArtosFlow": name => box[
  #box(image(
    "logo.svg",
    height: 0.7em,
  ))
  #name
]

This report is embedded in the
ArtosFlow project. ArtosFlow is a
project of the Artos Institute.
```

![](./images/img_20260115_051938.jpg)

위의 예제는 새로운 구문이 많이 등장합니다: `show` 키워드를 작성한 후 다르게 표시할 텍스트 문자열과 콜론을 붙입니다. 그런 다음 실제로 표시할 내용을 인자로 받는 함수를 작성합니다. 여기서 해당 인자를 `name` 이라 명명했습니다. 이제 함수 본체에서 `name` 변수를 사용해서 "ArtosFlow" 이름으로 출력할 수 있습니다. 지금의 쇼 규칙은 프로젝트 이름 앞에 로고 이미지를 추가하고 로고와 이름 사이에 줄바꿈이 발생하지 않도록 박스(`#box`) 함수 안에 배치합니다. 이미지도 박스 함수 안에 넣어 별도의 단락으로 표시되지 않도록 처리합니다.

첫 번째 박스 함수와 이미지 함수에 대한 함수 호출은 마크업에 포함되지 않기 때문에 선행된 `#` 문자는 필요하지 않습니다. Typst가 마크업 대신 코드를 기대할 때는 함수나 키워드 및 변수에 접근하기 위해 선행된 `#` 문자는 필요하지 않습니다. 이것은 매개변수 목록, 함수 정의 및 코드 블록에서도 관찰할 수 있습니다.


## Review

## The conference guidelines
학술회 웹사이트에서 레이아웃 가이드라인을 확인할 수 있습니다. 함께 살펴보겠습니다:

- 글꼴은 11pt 세리프체로 표시합니다.
- 제목은 17pt 볼드체로 표시합니다.
- 논문은 단일 열로 구성된 초록과 이중 열로 구성된 본문으로 구성합니다.
- 초록은 중앙 정렬합니다.
- 본문은 양쪽 정렬합니다.
- 1단계 섹션 제목은 13pt, 중앙 정렬, 소문자형 대문자로 표시합니다.
- 2단계 섹션 제목은 들여쓰기 처리하고 이탤릭체로 본문과 동일한 크기입니다.
- 마지막으로, 페이지는 US-레터 사이즈이며 각 페이지 하단 중앙에는 페이지 번호가 매겨지고 오른쪽 상단 모서리는 논문의 제목이 포함되야 합니다.

우리는 이미 이런 것들 중 많은 부분을 처리할 줄 알지만 일부는 새로운 방법을 배워야 합니다.


## Writing the right set rules
학술회 논문 문서에 사용할 몇 가지 기본 세트 규칙부터 작성하겠습니다.

```
#set page(
  paper: "us-letter",
  header: align(right)[
    A Fluid Dynamic Model for
    Glacier Flow
  ],
  numbering: "1",
)
#set par(justify: true)
#set text(
  font: "Libertinus Serif",
  size: 11pt,
)

#lorem(600)
```

![](./images/img_20260115_052054.jpg)

여기서 진행한 대부분의 작업은 이미 익숙합니다. 텍스트 크기를 11pt 로 설정하고 글꼴은 Libertinus Serif 로 지정했습니다. 또한 단락 정렬 기능을 활성화하고 페이지 크기를 US-레터로 설정했습니다.

페이지 세트 규칙에 사용한 헤더(`header`) 인수는 새로 추가된 기능입니다: 이를 통해 모든 페이지의 상단 여백을 채울 내용을 제공할 수 있습니다. 헤더는 학회 스타일 가이드의 요구사항에 따라 논문 제목을 명시합니다. 또한 논문 제목을 오른쪽 정렬하기 위해 `align` 함수를 사용합니다.

마지막으로 중요한 것은 페이지 번호 매기기(`numbering`) 옵션입니다. 여기서 페이지 번호의 패턴을 설정할 수 있습니다. “1” 로 설정하면 Typst는 페이지 번호만 표시합니다. “(1/1)” 로 설정하면 현재 페이지와 전체 페이지 수가 괄호로 표시됩니다. 심지어 완전히 사용자 정의된 함수로 원하는대로 형식을 지정할 수 있습니다.


## Creating a title and abstract
Now, let's add a title and an abstract. We'll start with the title. Typst comes with a title function. Let's start by providing our title as an argument:

이제 타이틀과 초록을 추가합니다. 타이틀부터 시작합니다. Typst에는 타이틀 함수가 포함되어 있습니다. 먼저 인수로 타이틀 문자열을 제공합니다:

```
#title[
  A Fluid Dynamic Model
  for Glacier Flow
]
```

![](./images/img_20260115_052120.jpg)

타이틀은 이미 굵게 표시되고 주변에 여백이 있음을 확인할 수 있습니다. 그러나 기본적으로 왼쪽 정렬되며 17pt 크기도 아닙니다. 따라서 타이틀의 외관은 조정해야 합니다. 타이틀 함수는 이렇게 설정할 수 있는 글꼴이나 텍스트 크기 관련 인수가 없습니다. 대신 이러한 속성은 텍스트 및 정렬 함수로 정의해야 합니다.

> [!NOTE] 타이틀 함수로 삽입한 내용과 등호(`=`)로 생성한 제목의 차이점은 무엇인가요?
>
> 제목은 문서에 여러 번 나타날 수 있지만(1단계 제목도 포함), 타이틀(title)은 일반적으로 문서 시작 부분에 한 번만 나타납니다. 이 둘을 구분해서 사용하면 Typst가 스크린 리더 같은 보조 기술 사용자에게 문서에 접근할 때 도움됩니다.

다른 요소 내부에 있는 특정 요소의 속성을 맞춤 설정하려면 `show-set` 규칙을 사용할 수 있습니다. 먼저 `show` 를 사용해서 맞춤 설정할 요소를 선택합니다. 이를 `선택자` (`selector`)라고 부릅니다. 그런 다음 콜론을 입력하고 선택자와 일치한 요소에 적용할 `set` 규칙을 작성합니다. 요약하면 구문은 다음과 같습니다:

```
#show your-selector: set some-element(/* ... */)
```

다시 정리하겠습니다: 타이틀을 중앙 정렬하고 17pt 크기로 설정하고자 합니다. 따라서 두 가지 `show-set` 규칙이 필요합니다:

- 하나의 선택자와 규칙 세트 텍스트 함수(글자 크기: 17pt)
- 하나의 선택자와 규칙 세트 정렬 함수(정렬: 중앙)

예제는 다음과 같습니다:

```
#show title: set text(size: 17pt)
#show title: set align(center)

#title[
  A Fluid Dynamic Model
  for Glacier Flow
]
```

![](./images/img_20260115_052229.jpg)

맞게 작성한 것 같습니다. 이제 저자 목록도 추가합니다: 지도 교수님과 함께 논문을 작성하므로 우리의 이름과 지도 교수님의 이름을 추가하겠습니다.

```
#grid(
  columns: (1fr, 1fr),
  align(center)[
    Therese Tungsten \
    Artos Institute \
    #link("mailto:tung@artos.edu")
  ],
  align(center)[
    Dr. John Doe \
    Artos Institute \
    #link("mailto:doe@artos.edu")
  ]
)
```

![](./images/img_20260115_052245.jpg)

두 저자에 대한 정보 블록을 나란히 배치합니다. 그리드(`grid`) 함수를 사용해서 이런 레이아웃을 생성합니다. 그리드 함수를 사용하면 각 열의 크기와 어떤 콘텐츠가 어떤 셀에 배치될지 정확히 제어할 수 있습니다. `columns` 인수는 상대적 길이 또는 분수 값의 배열을 인자로 받습니다. 이 경우 동일한 비율로 전달해서 사용 가능한 공간을 두 개의 동일한 열로 분할하도록 지시했습니다. 그런 다음 그리드 함수에 두 개의 콘텐츠 인수로 전달했습니다. 첫 번째는 우리의 정보, 두 번째는 지도 교수님의 정보입니다. 열의 콘텐츠를 중앙 정렬하기 위해 다시 정렬 함수를 사용했습니다. 그리드 함수는 셀을 지정하는 콘텐츠 인수를 임의의 개수만큼 받습니다. 행은 자동으로 추가되지만 행 인수를 사용해서 수동으로 크기를 조정할 수 있습니다.

저자와 타이틀이 너무 가깝게 배치되어 있습니다. 타이틀 아래에 공간을 확보하려면 다른 `show-set` 규칙을 사용합니다. 타이틀, 그리드, 단락 및 Typst가 페이지 상단부터 하단까지 배치하는 모든 요소를 **블록** (`block`)이라 합니다. 각 블록은 블록 함수로 제어합니다. 블록 간의 거리나 페이지 분할 여부 같은 동작을 제어합니다. 즉, 타이틀 선택자로 블록 간격을 설정하는 또 다른 `show-set` 규칙을 작성합니다:

```
#show title: set text(size: 17pt)
#show title: set align(center)
#show title: set block(below: 1.2em)

#title[
  A Fluid Dynamic Model
  for Glacier Flow
]

#grid(
  // ...
)
```

![](./images/img_20260115_052312.jpg)

위의 `show-set` 규칙으로 타이틀 아래의 간격을 재정의했습니다. 여기서 em 단위를 사용했는데 이것은 글꼴 크기의 배수로 표현합니다. 여기서 타이틀과 저자 목록 사이를 정확히 글꼴 크기의 1.2배 간격으로 설정했습니다. 이제 초록을 추가합니다. 학회에서 초록은 불규칙 정렬로 중앙 정렬한다는 점을 기억하시기 바랍니다.

```
...

#align(center)[
  #set par(justify: false)
  *Abstract* \
  #lorem(80)
]
```

![](./images/img_20260115_052330.jpg)

잘하셨습니다! 여기서 주목할 점은 초록은 정렬(`align`) 함수의 인수 내에서 특정 세트 규칙을 사용해서 정렬 기능을 해제했다는 것입니다. 이것은 첫 번째 세트 규칙 이후에 지정했음에도 문서의 나머지 부분에는 영향을 미치지 않습니다. **콘텐츠 블록(`[...]`)은 스타일링 범위를 제한** 하기 때문입니다. 콘텐츠 블록 내에서 설정된 사항은 해당 컨텐츠 블록 내의 콘텐츠에만 영향을 미칩니다.

또 다른 개선점은 헤더와 다이틀 요소(title element)의 인자 중복을 제거한 것입니다. 이 둘은 동일한 내용을 공유하므로 문서에 대한 메타데이터를 저장하도록 설계된 위치에서 이를 저장하는 것이 편리합니다. 그러면 두 위치에서 제목 문자열 검색할 수 있는 방법이 필요합니다. 문서 요소(document element)가 전자를 해결하는 데 도움이 될 수 있습니다: 세트 규칙(`set rule`)에서 이를 사용함으로 타이틀, 설명, 키워드 같은 문서의 메타데이터를 저장할 수 있습니다.

```
#set document(title: [A Fluid Dynamic Model for Glacier Flow])
```

PDF를 내보낼 때 여기에 설정된 제목 문자열은 PDF 리더 프로그램의 제목 표시줄에 표시됩니다. 운영 체제도 이 제목을 사용해서 파일을 검색 가능하게 합니다. 마지막으로 이 제목은 문서 접근성을 높이는 데 기여하며 접근성에 중점을 둔 PDF 표준인 PDF/UA 표준을 선택할 경우엔 필수 사항입니다.

이제 메인 타이틀과 헤더에 설정한 값을 가져올 방법이 필요합니다. `title` 함수는 문서 요소와 함께 작동하도록 설계됐기 때문에 인수를 지정하지 않고 호출하면 단순히 제목만 출력합니다. 헤더의 경우는 좀 더 명시적으로 처리해야 합니다: Typst가 제목을 거기에 삽입하길 원한다는 사실을 알 수 없기 때문에 수동으로 그렇게 하도록 지시해야 합니다.

컨텍스트(`context`)를 사용하면 요소에 설정한 모든 값의 내용을 가져올 수 있습니다. 컨텍스트 키워드를 사용하면 문서 요소의 `title` 속성을 포함해서 모든 요소의 모든 속성에 접근할 수 있습니다. 사용법은 다음과 같습니다:

```
#set document(title: [
  A Fluid Dynamic Model
  for Glacier Flow
])
...
#set page(
  header: align(
    right + horizon,
    // Retrieve the document
    // element's title property.
    context document.title,
  ),
  ...
)
#title()
...
```

![](./images/img_20260115_052407.jpg)

우선, 타이틀 함수를 빈 괄호로 호출한 점을 주목하시기 바랍니다. 인수가 전달되지 않았기 때문에 위에서 문서 요소에 설정한 값이 기본값으로 적용됩니다. **빈 괄호와 빈 대괄호의 차이는 중요** 합니다: **빈 괄호는 아무것도 전달하지 않음** 을 나타내는 반면 **빈 대괄호는 하나의 인수를 전달함** 을 의미합니다. 즉, 빈 콘텐츠 블록을 전달하는 것입니다. 그렇게 호출되면 타이틀에 표시되는 콘텐츠가 없게 됩니다.

다음으로 헤더를 살펴보겠습니다. 괄호 안의 타이틀 대신 컨텍스트 키워드를 사용해서 문서 제목에 접근했습니다. 이로 인해 위에서 설정한 내용이 정확히 삽입됩니다. 컨텍스트의 역할은 속성 접근에 국한되지 않습니다: 이를 통해 문서에 특정 요소가 존재하는지 확인하거나 다른 요소의 물리적인 크기를 측정하는 등의 다양한 작업을 수행할 수 있습니다. 컨텍스트를 활용하면 최종 사용자의 선호도에 반응하는 강력한 템플릿을 구축할 수 있습니다.

> [!NOTE] 왜 요소의 속성에 접근하려면 컨텍스트 키워드가 필요할까요?


## Adding columns and headings
위의 문서는 안타깝게 납으로 된 벽처럼 답답하게 보입니다. 이를 해결하기 위해 제목을 추가하고 전체 문서를 두 열의 레이아웃으로 전환하겠습니다. 다행히 이것은 매우 간단합니다: 페이지 세트 규칙에 열 수를 인수로 추가하면 됩니다.

컬럼 인수 목록에 2열을 추가함으로써 전체 문서를 두 열로 감쌌습니다. 그러나 이것은 제목과 저자 항목에도 영향을 미칩니다. 이들은 전체 페이지에 걸쳐 표시하려면 함수 호출을 통해 배치할 수 있습니다. `place` 함수는 정렬 방식과 배치할 내용을 위치 인수로 기대합니다. 명명된 범위 인수를 사용하면 항목을 현재 열에 대해 상대적으로 배치할지, 아니면 상위 요소(페이지)에 상대적으로 배치할지 결정할 수 있습니다. 한 가지 더 설정할 사항이 남아 있습니다: 만약 다른 인수를 제공하지 않으면 `place` 함수는 콘텐츠를 문서 흐름에서 분리해서 컨테이너 내의 다른 콘텐츠의 레이아웃에 영향을 주지 않고 그 위에 배치합니다:

```
#place(
  top + center,
  rect(fill: black),
)
#lorem(30)
```

![](./images/img_20260115_052444.jpg)

여기서 `place` 함수를 사용하지 않았다면 사각형은 별도의 줄에 독립적으로 위치했을 것입니다. 그러나 여기서는 사각형이 그 뒤에 오는 몇 줄의 텍스트와 겹칩니다. 마찬가지로 해당 텍스트는 사각형이 없는 것처럼 처리됩니다. 이런 동작을 변경하려면 `float: true` 인수를 전달해서 배치된 항목이 페이지 상단 또는 하단에서 차지하는 공간이 다른 콘텐츠에 의해 점유되지 않도록 처리할 수 있습니다.

```
#set page(
  paper: "us-letter",
  header: align(
    right + horizon,
    context document.title,
  ),
  numbering: "1",
  columns: 2,
)

#place(
  top + center,
  float: true,
  scope: "parent",
  clearance: 2em,
)[
  ...

  #par(justify: false)[
    *Abstract* \
    #lorem(80)
  ]
]

= Introduction
#lorem(300)

= Related Work
#lorem(200)
```

![](./images/img_20260115_052500.jpg)

이 예제에는 `v` 함수 대신 `place` 함수의 `clearance` 인자를 사용해서 본체와 객체 사이의 간격을 지정했습니다. 또한 배치 위치에서 중앙 정렬 기능은 상속되므로 각 구성 요소에 대한 명시적인 `align(center, ..)` 함수 호출을 제거할 수 있습니다.

이제 남은 작업은 단 하나 입니다: 제목을 스타일링하는 것입니다. 제목은 중앙 정렬하고 소문자형 대문자(small capitals)를 사용해야 합니다. 이 속성은 `heading` 함수에 기본적으로 제공되지 않으므로 몇 가지 `show-set` 규칙과 `show 규칙` 으로 직접 작성해야 합니다:

- 제목을 중앙 정렬하도록 설정하는 쇼 세트 규칙
- 제목을 13pt 크기로 만들고 일반 두께를 사용하도록 설정하는 쇼 세트 규칙
- 제목을 소문자 호출로 감싸도록 설정하는 쇼 세트 규칙

```
#show heading: set align(center)
#show heading: set text(
  size: 13pt,
  weight: "regular",
)
#show heading: smallcaps

...

= Introduction
...

== Motivation
...
```

![](./images/img_20260115_052525.jpg)

멋지네요! 모든 제목에 적용되는 쇼 세트 규칙을 사용했습니다. 마지막 쇼 세트 규칙에는 제목 전체에 소문자형 대문자 기능을 적용했습니다. 다음 예제에서 보게 될 것처럼 제목의 기본 모양을 완전히 재정의하는 사용자 지정 규칙을 제공할 수 있습니다.

지금 남은 유일한 문제는 "모든 제목이 동일하게 보인다" 는 점입니다. “Motivation” 와 “Problem Statement” 소제목은 기울임체로 처리된 연속 헤더라야 하지만 현재는 섹션 제목과 구분되지 않습니다. 이 문제를 해결하려면 `show` 규칙에 `where` 선택자를 적용해야 합니다: 이것은 헤더(및 기타 요소)에 적용할 수 있는 메서드로 속성에 따라 필터링할 수 있게 해줍니다. 이를 통해 섹션 제목과 소제목을 구분할 수 있습니다:

```
#show heading.where(level: 1): set align(center)
#show heading.where(level: 1): set text(size: 13pt, weight: "regular")
#show heading.where(level: 1): smallcaps

#show heading.where(level: 2): set text(
  size: 11pt,
  weight: "regular",
  style: "italic",
)
#show heading.where(level: 2): it => {
  it.body + [.]
}
```

![](./images/img_20260115_052546.jpg)

이 예제는 먼저 `.where(level: 1)` 구문을 사용해서 선택자를 좀 더 구체화함으로써 이전 규칙의 적용 범위를 1단계 헤딩으로 제한합니다. 그런 다음 두 번째 헤딩 레벨에 대한 쇼 세트 규칙을 추가합니다. 마지막으로 사용자 정의 함수를 포함한 `show` 규칙이 필요합니다: 기본적으로 제목은 내용을 블록으로 감쌉니다. 이로 인해 제목은 별도의 줄에 위치합니다. 그러나 우리는 제목이 본문과 자연스럽게 이어지도록 만들고 싶으므로 이 블록을 제거하기 위해 자체적인 `show` 규칙을 제공해야 합니다.

우리는 헤딩을 매개변수로 받는 함수를 규칙에 제공합니다. 이 매개변수는 관례적으로 "it" 라고 부르지만 다른 이름을 가질 수 있습니다. 이 매개변수는 콘텐츠로 사용될 수 있으며 기본 헤딩 전체를 그대로 표시합니다. 또는 자체 헤딩을 구성하려면 `body, numbering, level` 같은 필드를 활용해서 맞춤 스타일로 만들 수 있습니다. 여기선 헤딩 본문에 마침표를 붙여 출력하고 내장된 `show` 규칙이 생성하는 블록은 생략합니다. `show` 규칙에서 `it.numbering` 을 명시적으로 사용하지 않았기 때문에 이 헤딩은 더 이상 헤딩 번호 지정의 세트 규칙에 반응하지 않습니다. 이와 같이 `show` 규칙을 작성하면서 문서의 사용자 정의 기능을 그대로 유지하려면 이런 필드도 고려해야 합니다.

정말 멋지네요! 우리는 1단계와 2단계 제목에 선택적으로 적용되는 쇼 세트 규칙을 작성했습니다. `where` 선택자를 사용해서 제목을 단계별로 필터링했습니다. 그런 다음 하위 섹션 제목을 런인으로 렌더링했습니다. 또한 하위 섹션 제목 끝에는 자동으로 마침표를 추가합니다.

학술회 스타일 가이드를 다시 한번 살펴보겠습니다:

- 글꼴은 11pt 세리프체로 표시합니다. ✓
- 제목은 17pt 볼드체로 표시합니다. ✓
- 논문은 단일 열로 구성된 초록과 이중 열로 구성된 본문으로 구성합니다. ✓
- 초록은 중앙 정렬합니다. ✓
- 본문은 양쪽 정렬합니다. ✓
- 1단계 섹션 제목은 13pt, 중앙 정렬, 소문자형 대문자로 표시합니다. ✓
- 2단계 섹션 제목은 들여쓰기 처리하고 이탤릭체로 본문과 동일한 크기입니다. ✓
- 마지막으로, 페이지는 US-레터 사이즈이며 각 페이지 하단 중앙에는 페이지 번호가 매겨지고 오른쪽 상단 모서리는 논문의 제목이 포함되야 합니다. ✓

이제 우리는 이 모든 스타일 가이드를 준수했으며 논문을 학회에 제출할 수 있습니다! 완성된 논문은 다음과 같습니다:

## Reusing data with variables
지난 장에서 문서 대부분의 내용은 수동으로 입력했습니다. 제3장에는 문서 요소와 컨텍스트를 활용해서 이런 중복을 줄이고 타이틀을 한 번만 입력했습니다. 하지만 실제 문서는 여러 번 반복되는 요소가 훨씬 더 많을 수 있습니다. 이런 반복되는 값을 한 번만 정의해야 하는 이유는 다음과 같습니다:

1. 나중에 변경하기 더 쉬워집니다.
1. 특정 항목을 사용한 모든 부분을 빠르게 찾을 수 있습니다.
1. 전체적으로 일관성을 유지하기 용이합니다.
1. 길거나 입력하기 어려운 반복 구간에는 짧은 변수명이 입력하기 더 편리합니다.

기존 워드 프로세서를 사용했다면 나중에 검색할 수 있는 자리 표시자를 사용했을 겁니다. 그러나 Typst는 **변수를 사용해서 콘텐츠를 보다 안전하게 저장** 하고 변수 이름으로 문서 전체에 재사용할 수 있습니다.

이전에 배운 요소 속성을 컨텍스트를 활용해서 재현한 기법이 항상 가장 적합한 것은 아닙니다: Typst의 내장 요소는 문서의 타이틀이나 설명 같은 의미론적인 속성 또는 텍스트 크기처럼 직접적인 조판 관련 사항에 중점을 둡니다.

예제로 Typst에 대한 발음 기호를 살펴보겠습니다. 발음 기호를 표기하는 가장 좋은 방법 중 하나는 국제 음성 기호(IPA)입니다. 하지만 일반 키보드에 없는 문자를 사용하기 때문에 IPA를 반복해서 입력하는 것은 매우 번거로울 수 있습니다. 따라서 **여러 번 참조할 수 있는 변수로 정의** 하겠습니다.

```
#let ipa = [taɪpst]
```
여기서 우리는 변수로 정의하기 위해 새로운 키워드 `let` 을 사용합니다. 이 경우에는 변수 이름에 "ipa" 를 지정합니다. 마지막으로 등호와 변수 값을 입력합니다. 이 값은 내용이기 때문에 대괄호로 묶여 있으며 이것은 내용을 입력 받는 함수를 호출하는 방식과 유사합니다. 즉, 이 구문은 “변수 ipa 에 값 [taIpst] 를 할당하라” 는 문장을 의미합니다.

이제 문서에서 설정한 변수를 사용할 수 있습니다:

```
#let ipa = [taɪpst]

The canonical way to
pronounce Typst is #ipa.

#table(
  columns: (1fr, 1fr),
  [Name], [Typst],
  [Pronunciation], ipa,
)
```

![](./images/img_20260115_052725.jpg)

예제로 알 수 있듯이 변수는 마크업(`#` 접두사 사용)과 함수 호출(이름만 입력) 내에서 모두 사용할 수 있습니다. 물론 변수 값을 변경하면 해당 변수가 사용된 모든 위치가 자동으로 변경됩니다. IPA와 일반 문장을 구분하기 위해 IPA 변수 이름은 이탤릭체로 표시합니다. 또한 관례상 IPA를 둘러싸는 슬래시(`/`) 문자도 사용합니다.

```
#let ipa = text(
  style: "italic",
)[/taɪpst/]

The canonical way to
pronounce Typst is #ipa.

#table(
  columns: (1fr, 1fr),
  [Name], [Typst],
  [Pronunciation], ipa,
)
```

![](./images/img_20260115_052746.jpg)

여기서 우리는 텍스트 함수를 호출하고 그 반환 값으로 변수에 할당했습니다. 함수를 호출하면 함수는 인수를 처리한 후 다른 값(종종 콘텐츠)을 반환합니다. 지금까지 이 튜토리얼에는 대부분의 함수를 마크업 내에서 직접 호출했습니다. 예를 들어: `#text(fill: red)[CRIMSON!]` 텍스트 함수 호출은 빨간색의 텍스트를 반환값으로 반환합니다. 마크업 내에 배치했기 때문에 반환값은 작성한 콘텐츠에 즉시 삽입합니다. 이처럼 변수를 사용하면 나중에 재사용하거나 다른 값과 조합하기 위해 값을 저장할 수 있습니다.

변수에  저장할 내용은 제한되지 않습니다: Typst가 인식하는 모든 데이터 유형을 저장할 수 있습니다. 이 튜토리얼 전반에 걸쳐 Typst의 내장 함수에 데이터를 전달할 때는 다양한 데이터 유형을 활용했습니다. 다음은 각 데이터 유형을 **변수에 할당** 하는 예제입니다:

```
// Content with markup inside
#let blind-text = [_Lorem ipsum_ dolor sit amet]

// Unformatted strings
#let funny-font = "MS Comic Sans"

// Absolute lengths (see also pt, in, ...)
#let mile = 160934cm

// Lengths relative to the font size
#let double-space = 2em

// Ratios
#let progress = 80%

// Integer numbers
#let answer = 42

// Booleans
#let truth = false

// Horizontal and vertical alignment
#let focus = center
```

다음 장에는 변수와 사용자 정의 함수를 활용해서 여러 문서에 재사용할 수 있는 템플릿을 구축하는 방법에 대해 배웁니다.


## A toy template
Typst에서 **템플릿은 문서 전체를 감싸는 함수** 입니다. 이를 구현할 방법을 알아보기 전에 먼저 여러분만의 함수를 작성하는 방법에 대해 복습하겠습니다. 함수는 여러분이 원하는 모든 작업을 수행할 수 있으니 마음껏 창의력을 발휘해 보시기 바랍니다!

```
#let amazed(term) = box[✨ #term ✨]

You are #amazed[beautiful]!
```

![](./images/img_20260115_052832.jpg)

이전 섹션과 비교하면 이 부분이 `let` 을 사용한 **변수 정의와 매우 유사하다** 는 점을 눈치챘을 것입니다. 이 직감은 정확합니다: **함수는 단지 또 다른 데이터 유형** 일 뿐입니다. 여기서 우리는 "amazed" 란 변수를 정의하고 단일 인자 "term" 을 입력 받아 반짝임으로 둘러싸인 "term" 을 컨텐츠로 반환하는 함수를 정의합니다. 또한 전체를 하나의 박스로 묶어 "term" 이 줄바꿈으로 인해 분리되지 않도록 처리했습니다. 특수 함수 정의 구문은 정의를 보다 짧고 가독성 있게 만들지만 일반 변수 정의 구문도 사용할 수 있습니다. (자세한 내용은 스크립팅 참조 문서 참조) 함수 정의 후에는 모든 내장 함수처럼 이 함수도 호출할 수 있습니다.

Typst에 포함된 수 많은 함수는 선택적으로 **명명된 매개변수** 를 가질 수 있습니다. 우리의 함수도 이를 가질 수 있습니다!! 텍스트의 색상을 선택할 수 있는 매개변수를 함수에 추가하겠습니다. 매개변수가 지정되지 않은 경우를 대비해서 기본 색상도 제공합니다.

```
#let amazed(term, color: blue) = {
  text(color, box[✨ #term ✨])
}

You are #amazed[beautiful]!
I am #amazed(color: purple)[amazed]!
```

![](./images/img_20260115_052845.jpg)

템플릿은 이제 "amazed" 같은 사용자 정의 함수로 문서 전체를 감싸는 방식으로 작성합니다. 하지만 문서 전체를 거대한 함수 호출로 감싸는 것은 매우 번거롭습니다! 대신 “모든 것” 을 처리할 수 있는 `show` 규칙을 사용하면 좀 더 깔끔한 코드로 동일한 결과를 얻을 수 있습니다. 이런 `show` 규칙을 작성하려면 `show` 키워드 바로 뒤에 콜론 문자를 넣고 함수를 작성합니다. 이 함수는 문서의 나머지 부분을 매개변수로 받습니다. 이 함수는 해당 콘텐츠로 어떠한 작업도 수행할 수 있습니다. "amazed" 함수는 단일 콘텐츠 인자로 호출할 수 있으므로 이름으로 직접 쇼 규칙에 전달합니다. 한번 시도해 보겠습니다:

```
#show: amazed
I choose to focus on the good
in my life and let go of any
negative thoughts or beliefs.
In fact, I am amazing!
```

![](./images/img_20260115_052900.jpg)

이제 전체 문서가 "amazed" 함수로 전달됩니다. 이것은 마치 그 함수로 감싼 것처럼 작동합니다. 물론 이것은 특히 유용하진 않지만 **설정 규칙 및 명명된 인자와 결합하면 매우 강력** 해질 수 있습니다.


## Embedding set and show rules
템플릿 일부에 `set` 및 `show` 규칙을 적용하려면 함수 내의 콘텐츠 블록에 `set` 및 `show` 키워드를 사용한 후 해당 콘텐츠 블록에 문서를 삽입할 수 있습니다.

```
#let template(doc) = [
  #set text(font: "Inria Serif")
  #show "something cool": [Typst]
  #doc
]

#show: template
I am learning something cool today.
It's going great so far!
```

![](./images/img_20260115_052923.jpg)

이전 장에서 알아본 것 같이 `set` 규칙은 해당 콘텐츠 블록 내의 모든 요소에 적용됩니다. `everything show` 규칙은 문서 전체를 템플릿 함수에 전달하므로 템플릿 내의 `text set` 규칙과 `string show` 규칙은 문서 전체에 적용됩니다. 이러한 지식을 활용해서 이전 장에서 작성한 논문의 본문 스타일을 재현한 템플릿을 만들어 보겠습니다.

```
#let conf(title, doc) = {
  set page(
    paper: "us-letter",
    header: align(
      right + horizon,
      title
    ),
    columns: 2,
    ...
  )
  set par(justify: true)
  set text(
    font: "Libertinus Serif",
    size: 11pt,
  )

  // Heading show rules.
  ...

  doc
}

#show: doc => conf(
  [Paper title],
  doc,
)

= Introduction
...
```

![](./images/img_20260115_052941.jpg)

여기서 사용한 코드 대부분은 이전 장에서 그대로 복사해서 붙여넣었습니다. 두 가지 차이점은 다음과 같습니다:

1. 우리는 모든 것을 `conf` 함수에 `everything show` 규칙을 사용해서 감쌌습니다. 이 함수는 몇 가지 `set` 및 `show` 규칙을 적용하고 마지막에 전달받은 내용을 출력합니다.
1. 또한 콘텐츠 블록(`[...]`) 대신 중괄호(`{...}`)로 둘러싼 **코드 블록** 을 사용했습니다. 이렇게 처리하면 모든 세트 규칙과 함수 호출 앞에 `#` 문자를 사용할 필요가 없습니다. 대신 코드 블록 내에 직접 마크업을 작성할 수 없습니다.

또한 타이틀이 어디서 오는지도 주목하시기 바랍니다: 이전에는 변수에 있었습니다. 이제 템플릿 함수의 첫 번째 매개변수로 전달합니다. 이를 위해 `everything show` 규칙에 클로저(이름이 없는 함수로 즉시 사용됨)로 전달했습니다. 이렇게 처리한 이유는 `conf` 함수가 타이틀과 본문이란 두 개의 위치 매개변수를 기대하지만 `show` 규칙은 본문만 전달하기 때문입니다. 따라서 문서의 타이틀을 설정하고 `show` 규칙의 단일 매개변수를 사용할 수 있도록 처리하는 새로운 함수 정의를 추가합니다.


## Templates with named arguments
이전 장의 논문에는 타이틀과 저자 목록이 있었습니다. 이 요소들도 템플릿에 추가합니다. 타이틀 외에 템플릿에서 저자 목록과 소속 기관, 논문 초록을 받아들일 수 있도록 처리합니다. 가독성을 유지하기 위해 이 요소들을 명명된 인자 형태로 추가합니다. 최종적으로 다음과 같이 작성합니다:

```
#show: doc => conf(
  title: [
    A Fluid Dynamic Model for
    Glacier Flow
  ],
  authors: (
    (
      name: "Theresa Tungsten",
      affiliation: "Artos Institute",
      email: "tung@artos.edu",
    ),
    (
      name: "Eugene Deklan",
      affiliation: "Honduras State",
      email: "e.deklan@hstate.hn",
    ),
  ),
  abstract: lorem(80),
  doc,
)

...
```

이제 새로운 템플릿 함수를 만들어 보겠습니다. 먼저 `title` 인수에 기본값을 추가합니다. 이렇게 처리하면 타이틀을 지정하지 않고도 템플릿을 호출할 수 있습니다. 또한 "authors" 와 "abstract" 매개변수에 빈 기본값을 가진 명명된 매개변수로 추가합니다. 다음으로 이전 장에서 "title", "abstract" 및 "authors" 를 생성한 코드를 템플릿으로 복사해서 고정된 세부 정보를 매개변수 형태로 대체합니다.

새로운 "authors" 매개변수는 "name", "affiliation", "email" 키를 가진 사전 배열을 기대합니다. 저자의 수는 임의로 결정될 수 있으므로 저자 목록에 필요한 열 수를 동적으로 결정합니다. 먼저 "authors" 배열에 `.len()` 메서드를 적용해서 저자 수를 파악합니다. 그 다음 열 수를 이 개수와 3 중에서 최소값으로 설정해서 절대열을 3 이상으로  생성하지 않도록 처리합니다. 저자가 3명 이상일 경우는 새로운 행으로 추가됩니다. 이를 위해 그리드 함수에 `row-gutter` 매개변수를 추가했습니다. 그렇지 않으면 행 간격이 너무 좁아질 수 있습니다. 사전에서 저자의 세부 정보를 추출하기 위해 필드 접근 구문을 사용합니다.

각 저자에 대한 정보는 그리드 함수의 인수로 제공합니다: 이때 배열의 `map` 메서드가 유용합니다. 이 메서드는 배열의 각 항목마다 호출되는 함수를 인자로 받습니다. 우리는 각 저자의 세부 정보를 형식화하고 콘텐츠 값을 포함한 새로운 배열을 반환하는 함수를 전달합니다. 이제 그리드 함수의 여러 인자로 사용하는 하나의 값 배열을 얻었습니다. 이때 스프레드 연산자를 사용합니다. 이 연산자는 배열을 입력 받아 각 요소를 함수의 별도 인자로 적용합니다.

템플릿 함수는 다음과 같습니다:

```
#let conf(
  authors: (),
  abstract: [],
  doc,
) = {
  // Set and show rules from before.
  ...

  place(
    top + center,
    float: true,
    scope: "parent",
    clearance: 2em,
    {
      title()

      let count = authors.len()
      let ncols = calc.min(count, 3)
      grid(
        columns: (1fr,) * ncols,
        row-gutter: 24pt,
        ..authors.map(author => [
          #author.name \
          #author.affiliation \
          #link("mailto:" + author.email)
        ]),
      )

      par(justify: false)[
        *Abstract* \
        #abstract
      ]

    }
  )

  doc
}
```


## A separate file
대부분의 경우 템플릿은 별도의 파일에 정의한 후 문서로 가져옵니다. 이렇게 처리하면 현재 작성 중인 메인 파일을 깔끔하게 유지할 수 있고 템플릿을 보다 쉽게 재사용할 수 있습니다. 파일 패널에서 더하기 버튼을 클릭해서 새로운 텍스트 파일을 생성하고 파일 이름은 `conf.typ` 로 지정합니다. `conf` 함수 정의를 새로운 파일로 모두 이동시킵니다. 이제 메인 파일에서 `show` 규칙 앞에 `import` 문을 추가해서 호출할 수 있습니다. `import` 키워드와 콜론 사이에 파일 경로를 명시하고 가져올 함수 이름을 지정합니다.

템플릿을 적용하기 위한 좀 더 우아한 방법은 함수에 `.with` 메서드를 사용해서 모든 명명된 인수를 미리 채우는 것입니다. 이렇게 처리하면 클로저를 명시적으로 작성하고 템플릿 목록 맨 아래에 컨텐츠 인자를 추가하는 작업을 피할 수 있습니다. Typst Universe에 있는 모든 템플릿은 이런 스타일의 함수 호출과 함께 작동하도록 설계되어 있습니다.

```
#import "conf.typ": conf

#set document(title: [
  A Fluid Dynamic Model for
  Glacier Flow
])

#show: conf.with(
  authors: (
    (
      name: "Theresa Tungsten",
      affiliation: "Artos Institute",
      email: "tung@artos.edu",
    ),
    (
      name: "Eugene Deklan",
      affiliation: "Honduras State",
      email: "e.deklan@hstate.hn",
    ),
  ),
  abstract: lorem(80),
)

= Introduction
#lorem(90)

== Motivation
#lorem(140)

== Problem Statement
#lorem(50)

= Related Work
#lorem(200)
```
